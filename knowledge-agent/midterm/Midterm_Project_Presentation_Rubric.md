# Midterm Project Presentation Rubric

**Course:** BYU IS/IT Capstone — AI-Augmented Software Development  
**Total Points:** 100 (weighted across three grading domains)

---

## Philosophy

We grade your **process**, not your product. A team with a simpler app and a strong, documented process will outscore a team with a polished product and no evidence of how they built it.

### Grading Philosophy: Process Adherence as a Lens

Process adherence is not a separate grading area. It is the lens through which all areas are evaluated.

Students are expected to demonstrate:

- Document-driven development  
- AI-augmented workflows  
- Systematic iteration  
- Structured debugging  

This is assessed within each technical and product area. Course material is required process, not optional suggestions.

## Casey’s Technical Process (45% of Final Grade)

Each area is scored out of **25 points**. Total: **100 points**

### PRD & Document-Driven Development
- PRD is clear enough to build from  
- Documents drive the coding process and serve as source of truth  
- Development follows PRD → plan → roadmap → implementation  
- Documents are living artifacts  
- Evidence of AI-assisted iteration

### AI Development Infrastructure
- AI folder pattern implemented (`context.md`, project docs)  
- Project structure supports AI workflows  
- MCP configured and working  
- Git workflow shows branching, meaningful commits, and PRs

### Phase-by-Phase Implementation
- Code built incrementally following roadmap phases  
- Roadmaps used as checklists with tasks checked off  
- Evidence of multi-session workflow (plan / implement / review)

### Structured Logging & Debugging
- Structured logging implemented and used  
- CLI test scripts exist and work  
- Test → log → fix loop followed with AI diagnosing and fixing issues

---

## Jason’s Product & System Design (45% of Final Grade)

Each area is scored out of **20 points**. Total: **100 points**

### System Understanding
- System diagram showing ecosystem  
- Elements and relationships clearly mapped  
- Larger system goal identified  
- Leverage points understood

### Problem Identification
- Clear problem statement with justification  
- Alternative problems considered  
- Due diligence performed  
- Falsifiability check performed

### Customer Focus
- Target customer clearly identified  
- Solution positioned as best alternative  
- Differentiation from competition shown  
- Multiple analytical lenses applied

### Success & Failure Planning
- Success defined with measurable indicators  
- Failure defined  
- Pivot plans documented

### Customer Interaction
- Evidence of engaging with real customers  
- Feedback influencing iteration  
- Awareness of customer influence direction

## Presentation Details & Format

### Midterm Presentations
- **Duration:** 15 minutes per team (plan ~20 minutes in practice)  
- **Schedule:** 3 lecture periods (~3 teams per period)  
- **Content Split:** 50% Product/System Design, 50% Technical Demonstration

### Requirements
- All team members must contribute  
- System design diagram required  
- Frame presentation as pitching to C-suite executives (Casey and Jason)
